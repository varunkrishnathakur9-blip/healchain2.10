# HealChain Experimentation and Result Analysis

## Purpose

This folder contains the analysis pipeline used to generate benchmark and reporting artifacts for HealChain.
It supports both:

- Real data extraction (from backend database + Ganache chain)
- Synthetic/demo analysis (for quick visualization and development)

---

## What Is Considered Genuine Output

For genuine benchmark output, always use:

```bash
cd experimentation_and_result_Analysis
python run_real_analysis.py
```

This workflow:

1. Clears previous generated output
2. Extracts real task metrics for `task_037` and `task_038`
3. Regenerates benchmark markdown and JSON reports
4. Creates stable report copies for direct reference

---

## Output Locations

After `run_real_analysis.py`, output is available at:

- `results/healchain_benchmark_report_<timestamp>.md`
- `results/healchain_benchmark_report_<timestamp>.json`
- `experimentation_results/reports/HealChain_Task_Execution_report.md`
- `experimentation_results/reports/HealChain_Task_Execution_report.json`

The `experimentation_results/reports/` files are stable paths intended for easy sharing and citation.

## Troubleshooting

### 1) Backend/DB not reachable

Real extraction depends on backend DB access. If unavailable, extraction can fail or fall back.
Ensure backend and database are running and accessible from this machine.

### 2) Chain/RPC not reachable

On-chain timestamps and tx lookups require your configured RPC endpoint (Ganache/local chain).

### 3) Encoding artifacts in markdown

If you see malformed symbols in old files, regenerate reports with `run_real_analysis.py`.

---

## Recommendation for Thesis/Defense Reporting

Use only the latest files generated by `run_real_analysis.py` as the source of truth for measured values.
If you include demo outputs, label them explicitly as synthetic.
